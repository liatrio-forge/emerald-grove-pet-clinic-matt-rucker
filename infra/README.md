# Infrastructure

OpenTofu infrastructure for the Emerald Grove Veterinary Clinic on AWS.

## Prerequisites

- [Mise](https://mise.jdx.dev/) installed (manages OpenTofu version)
- AWS CLI configured with admin credentials (`aws configure`)
- Run `mise install` from the project root to install OpenTofu

## Directory Structure

```text
infra/
├── bootstrap/          # One-time state backend (S3 + DynamoDB)
├── modules/
│   ├── networking/     # VPC, subnets, security groups, ALB
│   ├── ecr/            # Container registry
│   ├── ecs/            # Fargate cluster, service, task definition
│   └── rds/            # PostgreSQL database
├── environments/
│   ├── staging.tfvars  # Staging-specific values
│   └── prod.tfvars     # Production-specific values
├── main.tf             # Root module (composes all modules)
├── variables.tf        # Input variables
├── outputs.tf          # Output values
└── providers.tf        # AWS provider + S3 backend config
```

## Bootstrap (One-Time Setup)

The state backend (S3 bucket + DynamoDB table) must be created before
any other infrastructure. This is a one-time operation.

```bash
# Initialize the bootstrap config
tofu -chdir=infra/bootstrap init

# Review what will be created
tofu -chdir=infra/bootstrap plan \
  -var='state_bucket_name=emerald-grove-tfstate' \
  -var='lock_table_name=emerald-grove-tflock'

# Create the state backend
tofu -chdir=infra/bootstrap apply \
  -var='state_bucket_name=emerald-grove-tfstate' \
  -var='lock_table_name=emerald-grove-tflock'
```

## Planning Infrastructure Changes

```bash
# Initialize (first time or after provider changes)
tofu -chdir=infra init

# Plan for staging
tofu -chdir=infra plan -var-file=environments/staging.tfvars

# Plan for production
tofu -chdir=infra plan -var-file=environments/prod.tfvars

# Or use the Mise shortcut (staging)
mise run tofu:plan
```

## Applying Changes

```bash
# Apply to staging
tofu -chdir=infra apply -var-file=environments/staging.tfvars

# Apply to production
tofu -chdir=infra apply -var-file=environments/prod.tfvars
```

## Environments

| Environment | tfvars File | Purpose |
|---|---|---|
| Staging | `environments/staging.tfvars` | Pre-production testing |
| Production | `environments/prod.tfvars` | Live application |

Both environments use the same root module with different variable values.

## Secrets

Application secrets are stored in AWS Secrets Manager:

- **Database password**: Auto-generated by OpenTofu, stored automatically
- **Anthropic API key**: Created as a placeholder — set the real value manually after first apply:

```bash
aws secretsmanager put-secret-value \
  --secret-id emerald-grove/staging/anthropic-api-key \
  --secret-string 'your-real-api-key'
```

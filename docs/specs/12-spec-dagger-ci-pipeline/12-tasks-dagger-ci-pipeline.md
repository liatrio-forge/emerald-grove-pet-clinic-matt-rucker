### Notes

- Use `podman` as the container runtime. Dagger may need `DOCKER_HOST` pointed at the Podman socket.
- Tests are run with `./mvnw test`. Pre-commit hooks remain active.
- Follow conventional commits: `feat(ci): ...` for pipeline code, `chore(ci): ...` for config.
- The Dagger module is initialized with `dagger init --sdk=go --name=ci --source=dagger`.
- Mise tool backends: `java` (core), `maven` (aqua), `node` (core), `dagger` (aqua).
